Hexo

SAE的速度实在太慢，而且域名解析需要到国外中转，于是把博客迁移到GitHub以后会督促自己更新。

什么是Hexo，为什么选择Hexo：

这次迁移和部署用到Hexo，hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github或者其他地方。有人对Hexo的评价是：

A fast, simple & powerful blog framework, powered by Node.js.

选择Hexo的原因很多，包括快速方便，支持Markdown等，它的作者是这样评价它的：

不可思议的快速 ─ 只要一眨眼静态文件即生成完成 支持 Markdown 仅需一道指令即可部署到 GitHub Pages 和 Heroku 已移植 Octopress 插件 高扩展性、自订性 兼容于 Windows, Mac & Linux

总结来说，它和github推荐的Jekyll相比，简化了很多git命令，对于一个博客应用来说，这是很有必要的。另外，它很小巧、利于理解，你可以很方便的修改原文件，创建属于你自己的主题等。

优化

关于Hexo的优化技巧，网上有很多，自行Google吧~

  • 用“多说”替换默认的评论

  • 添加“关于”页面
  • 添加“RSS”
  • 添加“sitemap”
  • 添加百度统计
